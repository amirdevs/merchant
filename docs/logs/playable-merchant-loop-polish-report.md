# Playable Merchant Loop Polish Report

Generated by `pnpm playtest:merchant-loop-polish`.

## Scope

This report records the intended manual pass for Phase 5: consequence visibility, balance readability, and UI polish around the first playable merchant loop.

## Manual checks

1. Run `pnpm build` and confirm the JSX arrow in the route card no longer breaks Vite/esbuild.
2. Open the Journal and inspect **Playable Merchant Loop v1**.
3. Confirm the top summary explains that Phase 5 is about balance and visible consequences.
4. Buy 4 Coastal Salt, travel to Riverwake Mill, and sell the salt.
5. Confirm **Balance Signals** updates money, profit, route, quest, and company readiness.
6. Confirm **Consequences** shows public trust, shadow heat, company readiness, and town reputation.
7. Resolve at least one quest with the honest approach and watch public trust rise.
8. Reset, run demo, and confirm most balance signals are healthy.
9. Try an exploit/shadow choice later and verify the design supports different consequence profiles.
10. Confirm the ledger still explains actions in story-friendly language.

## Acceptance criteria

- The build passes.
- The route card displays a safe arrow between towns.
- Players can see what to do next without reading developer notes.
- Consequences are visible before the game expands to more towns and questlines.
- Balance signals identify whether the vertical slice is healthy, risky, or still unfinished.

## Next expansion after Phase 5

The next large phase should move the loop closer to the global GameState save/export model and then expand content breadth: more towns, more rich questlines, more company features, and more route events.
